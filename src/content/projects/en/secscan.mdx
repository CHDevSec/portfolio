## The problem

Running Snyk against a fleet of microservices means running the CLI by hand for every repository, every branch, every pipeline. Engineers either skipped scans or copy-pasted shell scripts that drifted between teams. There was no consistent way to scan dozens of Go services at scale, no batch mode, and no clean integration with GitLab project IDs without writing custom glue every time.

## The approach

Built SecScan in Go — a CLI orchestrator that wraps Snyk CLI with three opinionated modes:

- **scanLocal** — recursively discovers `go.mod` files in a workspace and scans each as a separate project
- **scanMonitor** — clones one or many repositories from a URL list and scans them ephemerally (cleans up after)
- **scanGitLab** — accepts a GitLab project ID, pulls metadata via API, and scans the repo with proper credentials

All three modes feed Snyk Monitor for centralized dashboards, support `.env` / YAML / CLI config precedence, and return CI-friendly exit codes for pipeline gating.

## Engineering details that matter

- **Hexagonal architecture** — clean separation between CLI layer (Cobra), adapters (Snyk CLI, Git), and domain logic
- **Argument sanitization** — explicit `--org=<value>` formatting prevents Snyk CLI parsing ambiguities (a real bug I hit in production)
- **Credential isolation** — secrets stay in env vars, never in args, never in temp files
- **Repository batching** — `--repo-file` accepts URL lists for parallel scanning via GNU parallel
- **Three CI templates included** — GitHub Actions, GitLab CI, Jenkins pipelines documented in README

## The impact

- Replaced ad-hoc Snyk shell scripts with a single binary used across the Go stack
- Standardized vulnerability scanning for every Go service in the fleet
- CI/CD-native — same binary runs locally for dev triage and in pipelines for gate enforcement
- Open-source under MIT, with full README, architecture diagrams, troubleshooting guide, and three CI integration templates

## What I learned

Wrapping a CLI looks trivial until you hit edge cases like Snyk's `--org=true` parsing bug, transient Git credentials, and the difference between "scan worked" and "scan worked AND registered in Snyk dashboard." The thin layer of abstraction is where the real engineering lives.
