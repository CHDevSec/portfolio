## O problema

Rodar Snyk contra uma fleet de microsserviços significa rodar o CLI à mão pra cada repo, cada branch, cada pipeline. Engenheiros ou pulavam scans ou copiavam shell scripts que viravam snowflakes entre times. Não tinha forma consistente de escanear dezenas de serviços Go em escala, sem modo batch e sem integração limpa com IDs de projeto GitLab sem escrever glue customizado toda vez.

## A abordagem

Construí SecScan em Go — um CLI orchestrator que envelopa o Snyk CLI com três modos opinativos:

- **scanLocal** — descobre recursivamente arquivos `go.mod` num workspace e escaneia cada um como projeto separado
- **scanMonitor** — clona um ou vários repositórios de uma lista de URLs e escaneia efêmero (limpa depois)
- **scanGitLab** — aceita ID de projeto GitLab, puxa metadata via API e escaneia o repo com credenciais corretas

Os três modos alimentam o Snyk Monitor pra dashboards centralizados, suportam precedência de config `.env` / YAML / CLI e retornam exit codes amigáveis pra gating em pipeline.

## Detalhes de engenharia que importam

- **Arquitetura hexagonal** — separação clean entre camada CLI (Cobra), adapters (Snyk CLI, Git) e lógica de domínio
- **Sanitização de argumentos** — formatação explícita `--org=<value>` previne ambiguidades de parsing do Snyk CLI (bug real que peguei em produção)
- **Isolamento de credenciais** — secrets ficam em env vars, nunca em args, nunca em arquivos temp
- **Batching de repos** — `--repo-file` aceita lista de URLs pra scanning paralelo via GNU parallel
- **Três templates de CI inclusos** — GitHub Actions, GitLab CI, Jenkins pipelines documentados no README

## O impacto

- Substituiu shell scripts ad-hoc de Snyk por um único binário usado no stack Go
- Padronizou scanning de vulnerabilidades pra todo serviço Go na fleet
- CI/CD-native — mesmo binário roda local pra triagem de dev e em pipelines pra enforcement de gate
- Open-source sob MIT, com README completo, diagramas de arquitetura, guia de troubleshooting e três templates de integração CI

## O que aprendi

Envelopar um CLI parece trivial até você bater em casos de borda como o bug de parsing `--org=true` do Snyk, credenciais Git transientes e a diferença entre "scan rodou" e "scan rodou E registrou no dashboard Snyk". A camada fina de abstração é onde a engenharia real mora.
